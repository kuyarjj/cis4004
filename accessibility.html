Accessibility Statement

This site aims to conform to WCAG 2.2 AA where possible. Pages use semantic HTML5 landmarks, labeled form controls, sufficient color contrast, visible focus states, and keyboard‑accessible navigation. Images include descriptive alternative text. If you encounter barriers, email YOUR.EMAIL@EXAMPLE.COM.

Tester Checklist

  1. Navigate only with the keyboard (Tab/Shift+Tab/Enter/Space). All interactive elements must be focusable and operable.
  2. Check color contrast against background (aim ≥ 4.5:1 for body text).
  3. Ensure lang attribute is present and page has a descriptive <title>.
  4. Validate HTML via the W3C validator and fix any errors/warnings.
  5. Use headings in logical order (one H1 per page; no skipping levels for layout).
  6. Form fields have associated <label> and required states.
